# 时间选择器修改计划

## 问题分析

当前预约页面的时间选择使用按钮网格的形式，用户希望改为下拉选择的方式。

## 当前实现

- 日期选择：使用原生 `<input type="date">`
- 时间选择：使用按钮网格，点击按钮选择时段

## 修改方案

### 1. 修改预约页面 (`frontend/src/pages/booking/index.tsx`)

**修改内容**：
- 将时段选择从按钮网格改为 `<select>` 下拉选择框
- 保持现有的日期选择功能不变
- 保持验证逻辑不变
- 保持 UI 风格一致

**具体修改**：
- 替换第 279-301 行的按钮网格代码
- 改为使用 `<select>` 元素渲染可用时段
- 为已选时段添加标识
- 为不可用时段添加禁用选项

### 2. 保持功能完整性

- 日期加载逻辑保持不变
- 时段验证逻辑保持不变
- 表单提交逻辑保持不变

## 文件修改清单

- `frontend/src/pages/booking/index.tsx` - 修改预约页面的时间选择方式

## 风险评估

- 低风险：只修改 UI 展示方式，不修改核心逻辑
- 保持与现有 API 的兼容性
- 不影响后端代码
