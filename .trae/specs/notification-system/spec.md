# 通知系统 - 产品需求文档

## 概述

- **摘要**：为个人网站实现完整的通知系统，包括站内通知和邮件通知，通过异步消息队列处理。
- **目的**：提升用户体验，确保重要事件及时通知到相关人员（管理员和求职者）。
- **目标用户**：网站管理员和预约面试的求职者。

## 目标

1. 实现站内通知功能，支持通知列表、标记已读、全部已读
2. 实现未读通知数量实时显示
3. 在关键业务场景（新预约、预约确认/拒绝）自动触发通知
4. 提供邮件通知作为补充通知渠道
5. 确保通知系统与现有业务流程无缝集成

## 非目标（范围外）

- 暂不实现实时 WebSocket 推送（可通过轮询或后续优化）
- 暂不实现邮件发送队列消费者（可在后续迭代添加）
- 暂不实现预约提醒通知（可在后续迭代添加）

## 背景与上下文

现有系统已实现预约功能，但缺乏通知机制。当新预约提交或预约状态变更时，管理员和求职者无法及时获知。通知系统将填补这一空白。

## 功能需求

- **FR-1**：通知数据模型与存储
- **FR-2**：通知列表查询 API
- **FR-3**：标记通知已读 API
- **FR-4**：全部标记已读 API
- **FR-5**：未读通知数量查询 API
- **FR-6**：新预约自动触发通知
- **FR-7**：预约确认/拒绝自动触发通知
- **FR-8**：前端通知列表页面
- **FR-9**：未读数量在管理后台显示

## 非功能需求

- **NFR-1**：通知查询响应时间 < 200ms
- **NFR-2**：通知创建异步处理，不阻塞主业务流程
- **NFR-3**：遵循项目现有的代码规范和设计规范
- **NFR-4**：所有 API 需身份验证（除公开端点外）

## 约束

- **技术**：使用 Go + Gin（后端）、React + TypeScript（前端）、PostgreSQL（数据库）
- **业务**：邮件服务可后续集成，第一阶段优先实现站内通知
- **依赖**：复用现有认证中间件、API 响应格式等基础设施

## 假设

1. 管理员邮箱可从配置中获取
2. 求职者邮箱在预约时提供
3. 现有预约流程已完整实现

## 验收标准

### AC-1：通知数据模型
- **给定**：数据库已就绪
- **当**：创建通知表
- **则**：表结构包含 id, type, title, content, is_read, related_id, created_at 字段，并包含相应索引
- **验证**：programmatic

### AC-2：获取通知列表
- **给定**：用户已登录且存在通知数据
- **当**：调用 GET /api/notifications
- **则**：返回分页的通知列表，按创建时间倒序排列
- **验证**：programmatic

### AC-3：标记单条通知已读
- **给定**：存在未读通知
- **当**：调用 PUT /api/notifications/:id/read
- **则**：该通知的 is_read 字段更新为 true
- **验证**：programmatic

### AC-4：全部标记已读
- **给定**：存在多条未读通知
- **当**：调用 PUT /api/notifications/read-all
- **则**：所有未读通知的 is_read 字段更新为 true
- **验证**：programmatic

### AC-5：获取未读数量
- **给定**：用户已登录
- **当**：调用 GET /api/notifications/unread
- **则**：返回未读通知的数量
- **验证**：programmatic

### AC-6：新预约触发通知
- **给定**：用户提交新预约
- **当**：预约创建成功
- **则**：自动创建一条 type 为 new_booking 的站内通知给管理员
- **验证**：programmatic

### AC-7：预约状态变更触发通知
- **给定**：存在待处理的预约
- **当**：管理员确认或拒绝预约
- **则**：自动创建相应类型的站内通知（confirmed/rejected）
- **验证**：programmatic

### AC-8：通知列表页面
- **给定**：用户访问管理后台通知页面
- **当**：页面加载
- **则**：显示通知列表，未读通知有视觉标识，支持标记已读和全部已读
- **验证**：human-judgment

### AC-9：未读数量显示
- **给定**：用户登录管理后台
- **当**：存在未读通知
- **则**：在导航栏或其他显眼位置显示未读数量
- **验证**：human-judgment
