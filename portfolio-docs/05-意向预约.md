# Step 5: 意向预约

## 5.1 功能概述

求职者在线预约面试，支持时段选择和冲突校验。管理后台可管理预约状态。

## 5.2 数据库表

```sql
-- 预约时段设置表
CREATE TABLE schedule_settings (
    id SERIAL PRIMARY KEY,
    weekday INT NOT NULL CHECK (weekday BETWEEN 1 AND 5),
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    UNIQUE(weekday, start_time)
);

-- 预约表
CREATE TABLE bookings (
    id SERIAL PRIMARY KEY,
    company_name VARCHAR(100) NOT NULL,
    company_location VARCHAR(100) NOT NULL,
    booking_date DATE NOT NULL,
    booking_time TIME NOT NULL,
    contact_email VARCHAR(255) NOT NULL,
    contact_phone VARCHAR(20),
    notes TEXT,
    status VARCHAR(20) DEFAULT 'pending'
        CHECK (status IN ('pending', 'confirmed', 'rejected', 'completed', 'cancelled')),
    reject_reason TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(booking_date, booking_time)
);

-- 索引
CREATE INDEX idx_bookings_date ON bookings(booking_date);
CREATE INDEX idx_bookings_status ON bookings(status);
CREATE INDEX idx_bookings_created_at ON bookings(created_at DESC);
CREATE INDEX idx_bookings_date_time ON bookings(booking_date, booking_time);
```

## 5.3 API设计

### 预约（公开）

```
GET    /api/bookings/slots?date=2026-05-10   # 获取可选时段
POST   /api/bookings                          # 提交预约
```

**GET /api/bookings/slots 响应：**
```json
{
  "date": "2026-05-10",
  "weekday": "周日",
  "is_available": false,
  "message": "仅支持工作日预约"
}
```

```json
{
  "date": "2026-05-11",
  "weekday": "周一",
  "slots": [
    { "time": "09:00", "available": true },
    { "time": "10:00", "available": false, "reason": "已预约" },
    { "time": "11:00", "available": true },
    { "time": "14:00", "available": true },
    { "time": "15:00", "available": true },
    { "time": "16:00", "available": false, "reason": "已预约" }
  ]
}
```

**POST /api/bookings 请求：**
```json
{

  "booking_date": "2026-05-11",

```

### 预约管理（需登录）

```
GET    /api/bookings?page=1&page_size=20&status=pending
GET    /api/bookings/:id
```

**PUT /api/bookings/:id/status 请求：**

## 5.4 预约流程

```
求职者选择日期 → 获取可选时段 → 选择时段 → 填写信息 → 提交
                                                           │
                                                           ▼
                                                    后端校验（三重保障）
                                                    ├─ 前端禁用已约时段
                                                    ├─ 后端查询数据库校验
                                                    └─ 数据库UNIQUE约束兜底
                                                           │
                                              ┌────────────┴────────────┐
                                              ▼                         ▼
                                        保存成功                  时间冲突
                                        status=pending            返回错误
                                              │
                                              ▼
                                       触发通知（Step 6）
                                       ├─ 站内通知管理员
                                       └─ 邮件通知管理员
```

```
                    ┌──────────┐

                         │

       ┌──────────┐          ┌──────────┐
       │  已确认  │          │  已拒绝  │
```


## 5.7 表单校验规则

| 字段 | 规则 | 错误提示 |
|-----|------|---------|
| 公司名称 | 必填，2-100字符 | "请输入公司名称" |
| 公司地点 | 必填，2-100字符 | "请输入公司地点" |
| 预约日期 | 必填，工作日 | "请选择工作日" |
| 预约时段 | 必填，可选时段 | "请选择可用时段" |
| 联系邮箱 | 必填，邮箱格式 | "请输入正确的邮箱地址" |
| 联系电话 | 选填，手机号格式 | "请输入正确的手机号码" |
| 备注 | 选填，最大500字符 | "备注不能超过500字符" |

## 5.8 防刷机制

| 措施 | 说明 |
|-----|------|
| IP限流 | 同一IP每小时最多提交5次预约 |
| 邮箱限流 | 同一邮箱每天最多提交3次预约 |
| 前端禁用 | 已约时段前端禁用选择 |
| 后端校验 | 提交时查询数据库确认时段可用 |
| 数据库约束 | UNIQUE(booking_date, booking_time)兜底 |

## 5.9 缓存策略

| Key | TTL | 说明 |
|-----|-----|------|
| `booking:slots:{date}` | 5分钟 | 某天的可预约时段 |
