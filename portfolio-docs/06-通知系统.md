# Step 6: 通知系统

## 6.1 功能概述

站内通知 + 邮件通知，通过消息队列异步处理。

## 6.2 数据库表

```sql
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,
    type VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    is_read BOOLEAN DEFAULT FALSE,
    related_id INT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_notifications_is_read ON notifications(is_read);
CREATE INDEX idx_notifications_created_at ON notifications(created_at DESC);
CREATE INDEX idx_notifications_type ON notifications(type);
```

## 6.3 API设计

```
GET    /api/notifications?page=1&page_size=20           # 获取通知列表（需登录）
PUT    /api/notifications/:id/read                      # 标记已读（需登录）
PUT    /api/notifications/read-all                      # 全部标记已读（需登录）
GET    /api/notifications/unread                        # 获取未读数量（需登录）
```

## 6.4 通知触发场景

| 场景 | type | 站内通知 | 邮件通知 | 接收人 |
|-----|------|---------|---------|--------|
| 新预约提交 | `new_booking` | ✅ | ✅ | 管理员 |
| 预约确认 | `confirmed` | ✅ | ✅ | 求职者 |
| 预约拒绝 | `rejected` | ✅ | ✅ | 求职者 |
| 预约提醒 | `reminder` | ✅ | ✅ | 双方 |

## 6.5 消息队列设计

```
业务触发 → 生成通知 → 投递到队列 → 消费者处理
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
       站内通知消费者    邮件消费者      日志消费者
       (写入数据库)     (发送邮件)      (记录日志)
```

**队列定义：**

| 队列名 | 用途 | 优先级 | 重试 |
|-------|------|--------|------|
| `notification:internal` | 站内通知 | 中 | 5次 |
| `email:booking` | 预约邮件 | 高 | 3次 |
| `email:notification` | 通知邮件 | 中 | 3次 |

## 6.6 邮件服务

### 推荐服务商

| 场景 | 服务商 |
|-----|-------|
| 国内用户 | 阿里云邮件推送 |
| 国际用户 | SendGrid |

### 邮件模板

**新预约通知（管理员）：**
```
主题：[个人网站] 收到新的面试预约

您好，

{company_name}（{company_location}）预约了面试：

时间：{booking_date} {booking_time}
联系人：{contact_email}
备注：{notes}

请登录管理后台查看详情并确认。
```

**预约确认通知（求职者）：**
```
主题：[个人网站] 面试预约已确认

{company_name}，您好！

您的面试预约已确认：

时间：{booking_date} {booking_time}

如有变更，请及时联系。
```
