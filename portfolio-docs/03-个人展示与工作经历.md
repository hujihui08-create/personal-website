# Step 3: 个人展示与工作经历

## 3.1 功能概述

首页展示个人信息、工作经历时间线、社交链接，支持简历下载。管理后台可编辑个人资料和工作经历。

## 3.2 数据库表

```sql
-- 个人资料表
CREATE TABLE profiles (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    title VARCHAR(100),
    bio TEXT,
    avatar_url VARCHAR(500),
    github_url VARCHAR(500),
    linkedin_url VARCHAR(500),
    email VARCHAR(100),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 工作经历表
CREATE TABLE work_experiences (
    id SERIAL PRIMARY KEY,
    company_name VARCHAR(200) NOT NULL,
    position VARCHAR(100) NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE,
    description TEXT NOT NULL,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 工作经历与项目关联表
CREATE TABLE experience_projects (
    id SERIAL PRIMARY KEY,
    experience_id INT NOT NULL REFERENCES work_experiences(id) ON DELETE CASCADE,
    project_id INT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    sort_order INT DEFAULT 0,
    UNIQUE(experience_id, project_id)
);

-- 索引
CREATE INDEX idx_experiences_sort_order ON work_experiences(sort_order DESC);
CREATE INDEX idx_experience_projects_experience ON experience_projects(experience_id);
CREATE INDEX idx_experience_projects_project ON experience_projects(project_id);
```

## 3.3 API设计

### 个人资料

```
GET    /api/profile                 # 获取个人资料（公开）
PUT    /api/profile                 # 更新个人资料（需登录）
POST   /api/profile/avatar          # 上传头像（需登录）
```

**GET /api/profile 响应：**
```json
{
  "name": "张三",
  "title": "高级前端工程师",
  "bio": "热爱技术，专注React/Go全栈开发",
  "avatar_url": "https://cdn.example.com/avatar.jpg",
  "github_url": "https://github.com/zhangsan",
  "linkedin_url": "https://linkedin.com/in/zhangsan",
  "email": "zhangsan@email.com",
  "skills": ["React", "TypeScript", "Go", "PostgreSQL", "Docker"]
}
```

### 工作经历

```
GET    /api/experiences             # 获取工作经历列表（公开，按时间倒序）
POST   /api/experiences             # 创建工作经历（需登录）
PUT    /api/experiences/:id         # 更新工作经历（需登录）
DELETE /api/experiences/:id         # 删除工作经历（需登录）
PUT    /api/experiences/reorder     # 调整排序（需登录）
```

**GET /api/experiences 响应：**
```json
{
  "data": [
    {
      "id": 1,
      "company_name": "ABC科技有限公司",
      "position": "高级前端工程师",
      "start_date": "2024-03-01",
      "end_date": null,
      "description": "负责公司核心产品的前端架构设计与开发...",
      "projects": [
        { "id": 1, "name": "电商平台重构", "type": "enterprise" },
        { "id": 2, "name": "数据中台系统", "type": "enterprise" }
      ]
    }
  ]
}
```

### 简历

```
GET    /api/resume                  # 获取简历下载链接（公开）
POST   /api/resume                  # 上传简历（需登录）
```

## 3.5 交互要求

- 页面加载：淡入动画
- 卡片悬停：轻微上浮阴影
- 简历下载：显示进度提示
- 工作经历：滚动渐入动画
- 关联项目标签：悬停显示项目名称，点击跳转详情
- 展开/收起：平滑过渡动画

## 3.6 缓存策略

| Key | TTL | 说明 |
|-----|-----|------|
| `profile:main` | 1小时 | 个人资料 |
| `experiences:list` | 1小时 | 工作经历列表 |
