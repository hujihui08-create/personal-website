## 9.3 Docker Compose（生产）

```yaml
# docker-compose.yml
version: '3.8'
services:
  frontend:
    image: portfolio-frontend:latest
    restart: always
    ports:
      - "3000:80"
    depends_on:
      - backend

  backend:
    image: portfolio-backend:latest
    restart: always
    ports:
      - "8080:8080"
    environment:
      - DB_HOST=postgres
      - DB_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=redis
      - MINIO_ENDPOINT=minio
      - JWT_SECRET=${JWT_SECRET}
    depends_on:
      - postgres
      - redis

  postgres:
    image: postgres:15-alpine
    restart: always
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=portfolio
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=${DB_PASSWORD}

  redis:
    image: redis:7-alpine
    restart: always
    volumes:
      - redis_data:/data

  minio:
    image: minio/minio
    restart: always
    command: server /data --console-address ":9001"
    volumes:
      - minio_data:/data

  nginx:
    image: nginx:alpine
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./nginx/ssl:/etc/nginx/ssl
    depends_on:
      - frontend
      - backend

volumes:
  postgres_data:
  redis_data:
  minio_data:
```

## 9.4 Nginx配置

```nginx
# nginx/nginx.conf
upstream frontend {
    server frontend:80;
}

upstream backend {
    server backend:8080;
}

server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # 安全头
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:;";

    # 静态资源缓存
    location /assets/ {
        proxy_pass http://frontend;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 前端路由
    location / {
        proxy_pass http://frontend;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # SSE代理（Agent流式响应）
    location /api/agent/chat {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }

    # 文件上传大小限制
    client_max_body_size 10M;
}
```

## 9.6 监控方案

### 工具链

| 层级 | 工具 | 用途 |
|-----|------|------|
| 基础设施 | Prometheus + Grafana | CPU、内存、磁盘、网络 |
| 应用性能 | Sentry | 错误追踪、性能分析 |
| 日志收集 | Loki + Grafana | 结构化日志查询 |
| 数据库 | pg_stat_statements | 慢查询分析 |
| 前端性能 | Lighthouse CI | 性能评分监控 |

### 告警规则

| 告警项 | 阈值 | 通知方式 |
|-------|------|---------|
| 服务宕机 | 不可用 > 1分钟 | 邮件 + 短信 |
| API错误率 | > 5% | 邮件 |
| API响应时间 | P95 > 1秒 | 邮件 |
| 数据库连接 | > 80% | 邮件 |
| 磁盘空间 | > 85% | 邮件 + 短信 |
| Redis内存 | > 80% | 邮件 |

## 9.8 备份策略

| 数据 | 频率 | 保留时间 | 方式 |
|-----|------|---------|------|
| PostgreSQL | 每日全量 | 30天 | pg_dump + 压缩 |
| MinIO文件 | 每日增量 | 90天 | rsync |
| Redis | 持久化RDB | 7天 | 自动 |
