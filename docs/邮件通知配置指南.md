# 邮件通知配置指南

## 概述

个人网站现在支持邮件通知功能！当有新预约或者预约状态变更时，系统会自动发送邮件通知管理员和用户。

## 免费邮件服务商推荐

### Brevo (原 Sendinblue) - 推荐 ⭐

- **免费额度**：每天 300 封邮件，永久免费
- **无需信用卡**
- **注册地址**：https://www.brevo.com/
- **邮件质量**：高，送达率好

**其他选项**：
- **SendGrid**：免费计划已取消，需付费
- **Mailgun**：每天 100 封免费
- **Amazon SES**：前 12 个月每月 3000 封免费

## Brevo 配置步骤

### 1. 注册 Brevo 账号

1. 访问 https://www.brevo.com/ 注册账号
2. 完成邮箱验证
3. 登录后台

### 2. 获取 API Key

1. 登录后，点击右上角用户名 → **SMTP & API**
2. 选择 **API Keys** 标签
3. 点击 **Generate a new API key**
4. 给 Key 起个名字（如：portfolio-website）
5. 复制生成的 API Key（只显示一次！）

### 3. 配置发件人

1. 在 Brevo 后台，进入 **Senders** 页面
2. 添加一个新的发件人
3. 填写发件人邮箱和名称
4. 完成验证

### 4. 更新项目配置

在项目根目录创建或编辑 `.env.development` / `.env.production` 文件：

```env
# 邮件服务配置
EMAIL_PROVIDER=brevo
EMAIL_API_KEY=你的-brevo-api-key
EMAIL_FROM=noreply@yourdomain.com
EMAIL_FROM_NAME=个人网站
EMAIL_ADMIN=admin@yourdomain.com
```

**配置说明**：
- `EMAIL_PROVIDER`：使用 brevo
- `EMAIL_API_KEY`：从 Brevo 获取的 API Key
- `EMAIL_FROM`：发件人邮箱（需在 Brevo 验证）
- `EMAIL_FROM_NAME`：发件人显示名称
- `EMAIL_ADMIN`：接收通知的管理员邮箱

## SMTP 配置（备选方案）

如果你想使用其他 SMTP 服务，配置如下：

```env
EMAIL_PROVIDER=smtp
EMAIL_SMTP_HOST=smtp.example.com
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USER=your-email@example.com
EMAIL_SMTP_PASSWORD=your-password
EMAIL_FROM=noreply@yourdomain.com
EMAIL_FROM_NAME=个人网站
EMAIL_ADMIN=admin@yourdomain.com
```

**常见 SMTP 服务器**：
- Brevo: `smtp-relay.brevo.com:587`
- Gmail: `smtp.gmail.com:587`
- Outlook: `smtp.office365.com:587`

## 通知场景

系统会在以下场景发送邮件：

### 1. 新预约通知（给管理员）

- **触发时机**：用户提交新预约时
- **接收人**：`EMAIL_ADMIN` 配置的邮箱
- **内容**：包含公司信息、预约时间、联系方式等

### 2. 预约确认通知（给用户）

- **触发时机**：管理员确认预约时
- **接收人**：预约时填写的联系邮箱
- **内容**：确认信息和预约时间

### 3. 预约拒绝通知（给用户）

- **触发时机**：管理员拒绝预约时
- **接收人**：预约时填写的联系邮箱
- **内容**：拒绝原因（如有）

## 禁用邮件通知

如果不想使用邮件通知，只需不配置相关环境变量即可。系统仍会正常工作，只是不会发送邮件，站内通知功能仍然可用。

## 测试邮件通知

1. 启动后端服务
2. 在前端创建一个新预约
3. 检查管理员邮箱是否收到通知
4. 在管理后台确认/拒绝预约
5. 检查用户邮箱是否收到通知

## 故障排查

### 邮件没有收到？

1. 检查垃圾邮件文件夹
2. 确认 API Key 配置正确
3. 检查后端日志是否有错误
4. 确认发件人邮箱已在 Brevo 验证

### 如何查看后端日志？

启动服务后，查看控制台输出，如果有邮件发送错误会显示相关信息。

## 注意事项

1. **不要提交 API Key**：确保 `.env` 文件在 `.gitignore` 中
2. **发件人验证**：使用 Brevo 时必须先验证发件人邮箱
3. **免费额度**：Brevo 免费计划每天 300 封，足够个人使用
4. **异步发送**：邮件在后台异步发送，不影响主流程
