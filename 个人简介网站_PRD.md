# 个人简介网站 PRD文档

## 一、产品概述

### 1.1 产品定位
一个集个人展示、作品管理、智能问答、求职预约于一体的现代化个人简介网站。面向求职者展示专业能力与项目成果，支持在线预约面试，同时为管理员提供便捷的内容管理与通知后台。

### 1.2 核心功能
| 模块 | 功能描述 |
|------|----------|
| 个人展示 | 个人介绍、简历下载 |
| 作品展示 | 企业项目+个人项目详情页 |
| 智能助手 | 基于RAG的问答Agent，支持语音输入 |
| 意向预约 | 求职者在线预约面试，含时间冲突校验 |
| 管理后台 | 登录后管理所有内容，接收预约通知 |

### 1.3 技术栈
- **前端**: React + TypeScript + shadcn/ui + Tailwind CSS
- **后端**: Go + Gin框架
- **数据库**: PostgreSQL
- **向量库**: pgvector (RAG知识库)
- **邮件服务**: SMTP / 第三方邮件API
- **部署**: 云服务器 + Docker

### 4.3 Agent助手模块

#### 4.3.1 界面设计

| 功能项 | 描述 |
|--------|------|
| 入口位置 | 首页底部固定区域或独立页面 `/agent` |

#### 4.3.2 功能特性

| 功能 | 说明 |
|------|------|
| 语音输入 | 点击麦克风，实时语音识别转文字 |

| 上下文记忆 | 支持多轮对话，保持上下文 |
| 推荐问题 | 首次进入展示3-5个推荐问题 |
| 语音朗读 | 可选：回答支持语音播放（TTS） |

#### 4.3.3 RAG知识库设计

| 数据源 | 内容 |
|--------|------|
| 个人介绍文档 | 详细版自我介绍、职业经历 |

**文档处理流程**:
```
原始文档 → 文本清洗 → 分块(Chunk) → Embedding → 存入pgvector
```

### 4.4 意向预约模块（求职者预约面试）

#### 4.4.1 前端预约表单

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 公司名称 | 文本输入 | 是 | 不超过100字符 |
| 公司地点 | 文本输入 | 是 | 不超过100字符 |
| 预约日期 | 日期选择器 | 是 | 仅可选择未来日期 |
| 预约时段 | 时间选择器 | 是 | 从可选时段中选择 |
| 联系邮箱 | 邮箱输入 | 是 | 合法邮箱格式 |
| 联系电话 | 电话输入 | 否 | 合法手机号/电话格式 |
| 备注信息 | 文本域 | 否 | 不超过500字符 |

#### 4.4.2 时间冲突校验机制

```
┌─────────────────────────────────────────────────────────────────┐
│                      时间冲突校验逻辑                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 管理员在后台设置可用时段                                      │
│     ┌───────────────────────────────────────────────────┐       │
│     │  工作日: 周一至周五                                  │       │
│     │  可选时段:                                          │       │
│     │    09:00-10:00  10:00-11:00  11:00-12:00          │       │
│     │    14:00-15:00  15:00-16:00  16:00-17:00          │       │
│     │  每个时段仅接受1个预约                               │       │
│     └───────────────────────────────────────────────────┘       │
│                                                                 │
│  2. 前端日期选择器                                               │
│     ├─ 自动禁用非工作日（周末、节假日）                           │
│     ├─ 选择日期后，动态加载该日可选时段                           │
│     └─ 已被预约的时段显示"已约"并置灰                            │
│                                                                 │
│  3. 后端二次校验（防止并发冲突）                                  │
│     ├─ 查询该时段是否已有预约记录                                │
│     ├─ 使用数据库行锁防止并发重复预约                             │
│     └─ 冲突时返回错误 + 可选时段列表                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| 待确认 | 新预约等待管理员处理 | 求职者提交预约 |
| 已确认 | 管理员确认面试时间 | 管理员点击确认 |
| 已拒绝 | 管理员拒绝预约 | 管理员点击拒绝 |
| 已完成 | 面试已完成 | 管理员标记完成 |
| 已关闭 | 预约已过期/取消 | 超时自动关闭或手动取消 |

#### 4.4.4 通知机制

| 触发事件 | 站内通知 | 邮件通知 |
|----------|----------|----------|
| 新预约提交 | ✅ 管理员收到通知 | ✅ 发送给管理员邮箱 |
| 预约确认 | ✅ 求职者可查看 | ✅ 发送给求职者邮箱 |
| 预约拒绝 | ✅ 求职者可查看 | ✅ 发送给求职者邮箱 |

**邮件模板内容**:
- **新预约通知（管理员）**: 求职者信息 + 公司名称 + 预约时间 + 查看链接
- **确认通知（求职者）**: 面试确认时间 + 公司信息 + 联系方式
- **拒绝通知（求职者）**: 拒绝原因（可选）+ 重新预约链接

## 八、数据库设计

### 8.1 表结构

```sql
-- 管理员表
CREATE TABLE admins (
    id SERIAL PRIMARY KEY,
    password_hash VARCHAR(255) NOT NULL,  -- bcrypt加密
    notification_email VARCHAR(255),       -- 接收通知的邮箱
    created_at TIMESTAMP DEFAULT NOW()
);

-- 个人资料表
CREATE TABLE profiles (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    title VARCHAR(100),
    bio TEXT,
    avatar_url VARCHAR(500),
    github_url VARCHAR(500),
    linkedin_url VARCHAR(500),
    email VARCHAR(100),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 作品表
CREATE TABLE projects (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    type VARCHAR(20) CHECK (type IN ('enterprise', 'personal')),
    start_date DATE,
    end_date DATE,
    summary TEXT,
    description TEXT,  -- 项目实现详情
    cover_image VARCHAR(500),
    images TEXT[],  -- 项目图片数组
    github_url VARCHAR(500),
    demo_url VARCHAR(500),
    tags VARCHAR(50)[],
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 预约时段设置表
CREATE TABLE schedule_settings (
    id SERIAL PRIMARY KEY,
    weekday INT NOT NULL CHECK (weekday BETWEEN 1 AND 5),  -- 1=周一, 5=周五
    start_time TIME NOT NULL,   -- 时段开始时间
    end_time TIME NOT NULL,     -- 时段结束时间
    is_active BOOLEAN DEFAULT TRUE,
    UNIQUE(weekday, start_time)
);

-- 预约表
CREATE TABLE bookings (
    id SERIAL PRIMARY KEY,
    company_name VARCHAR(100) NOT NULL,
    company_location VARCHAR(100) NOT NULL,
    booking_date DATE NOT NULL,
    booking_time TIME NOT NULL,
    contact_email VARCHAR(255) NOT NULL,
    contact_phone VARCHAR(20),
    notes TEXT,
    status VARCHAR(20) DEFAULT 'pending'
        CHECK (status IN ('pending', 'confirmed', 'rejected', 'completed', 'cancelled')),
    reject_reason TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(booking_date, booking_time)  -- 同一时间段仅一个预约
);

-- 通知记录表
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,
    type VARCHAR(50) NOT NULL,          -- new_booking / confirmed / rejected
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    is_read BOOLEAN DEFAULT FALSE,
    related_id INT,                     -- 关联的预约ID
    created_at TIMESTAMP DEFAULT NOW()
);

-- 知识库文档表
CREATE TABLE knowledge_docs (
    id SERIAL PRIMARY KEY,
    filename VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    embedding VECTOR(1536),  -- OpenAI embedding维度
    created_at TIMESTAMP DEFAULT NOW()
);

-- Agent会话表
CREATE TABLE chat_sessions (
    id SERIAL PRIMARY KEY,
    session_id VARCHAR(100) UNIQUE NOT NULL,
    messages JSONB DEFAULT '[]',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```
